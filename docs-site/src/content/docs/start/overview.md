이 사이트는 **WP-TT 프로젝트의 단일 출처(single source of truth)** 입니다. 컨테이너 터미널 TT(트랜스퍼 트럭)의 모니터링·배차·사이클 분석과 AI 배차 연구에서 알아낸 것을 한곳에 쌓아, **"예전에 알아낸 걸 다시 까먹는" 손실을 막는 것**이 목적입니다.

검색(전문 검색 Pagefind 내장), 자동 사이드바, 일관된 컨벤션으로 — 흩어진 메모·코드 주석·세션 인수인계 파일을 대체합니다.

## 어떻게 쓰나 (요약)

- 새 글은 알맞은 섹션 디렉토리에 마크다운(`.md`/`.mdx`)으로 추가하면 **사이드바에 자동 등록**됩니다.
- 글을 쓰기 전에 [글쓰기 규칙](/kc/start/contributing/)을 확인하세요 — 프론트매터·상태표시·링크 규칙이 모든 글에 일관 적용됩니다.
- 리서치·ADR·실험처럼 **사실 정확성이 중요한 글**은 본문 첫 줄에 상태·검토일을 답니다.
- **검증 안 된 수치·식별자·API 동작은 단정하지 말고 `(미검증)`으로 표기**하고 "열린 질문"에 남깁니다.

:::note[이 사이트는 정적 빌드]
`npm run build`로 빌드된 정적 사이트를 운영 API가 `/kc/`로 서빙합니다. 내부 링크는 그래서 `/kc/...` 절대경로를 씁니다([글쓰기 규칙](/kc/start/contributing/) 참고).
:::
