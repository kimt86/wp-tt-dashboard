이 규칙은 **모든 글에 일관되게** 적용됩니다. 새 글을 쓰기 전에 먼저 읽으세요.

## 파일·제목

- **파일명은 영문 kebab-case** (`cycle-detection.md`). 화면 **제목은 한국어**.
- 글은 알맞은 섹션 디렉토리에 두면 사이드바에 자동 등록됩니다(슬러그 = 파일 경로).

## 프론트매터

모든 문서 상단에:

```yaml
---
title: 한국어 제목          # 필수
description: 한 줄 설명       # 권장 (검색·미리보기)
sidebar:
  order: 1                  # 섹션 내 정렬(작을수록 위). 생략 시 알파벳순
---
```

## 상태표시 (리서치 · ADR · 실험)

사실 정확성이 중요한 글은 **본문 첫 줄**에 상태와 최종 검토일을 답니다:

```markdown
**상태:** `검증됨` · **최종 검토:** 2026-06-12
```

- 상태 값: `초안` · `검증됨` · `폐기됨`
- 날짜는 `YYYY-MM-DD`.

## ★ 사실 원칙 — 추측을 사실처럼 쓰지 않기

이 프로젝트의 가장 비싼 자산은 **검증된 지식**입니다. 그래서:

- **검증 안 된 구체 수치·식별자·API 동작은 단정하지 않는다.** `(미검증)`으로 표기하고 글 끝의 **"열린 질문"** 절에 남긴다.
- 관측 못 한 값은 비워 둔다(NULL). 그럴듯한 추정으로 채우지 않는다.
- 외부 사실(API 명세·수치)은 **1차(공식) 출처**를 확인하고 출처를 남긴다. 모르면 "모른다"고 적는다.
- 정정·번복도 지운지 말고 기록한다([구현 기록](/kc/journal/)·[실험](/kc/experiments/)).

:::caution[왜 이렇게까지]
"그럴듯하지만 틀린" 사실 하나가 나중에 몇 시간을 잡아먹습니다. 불확실하면 `(미검증)`이 정직하고 더 빠릅니다.
:::

## 링크

- 내부 문서는 **`/kc/...` 절대경로**로 상호 링크합니다. 예: `[개요](/kc/start/overview/)`.
- 이 사이트는 base가 `/kc`이므로(운영 API가 `/kc/`로 서빙) 링크에 `/kc` 접두사가 필요합니다. 접두사를 빼면 대시보드 SPA로 가버려 깨집니다.

## 문체

- **결론을 먼저, 근거는 나중.** 군더더기 없이.
- 한국어로 쓰되, 기술 용어는 처음 등장 시 한글(영문)처럼 병기. 예: 트랜스퍼 트럭(TT), 의사결정 기록(ADR).
- 표·코드펜스·[Starlight 컴포넌트](https://starlight.astro.build/components/asides/)(`:::note` 등)를 적극 활용.
