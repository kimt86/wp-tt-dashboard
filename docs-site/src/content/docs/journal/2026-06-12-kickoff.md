## 한 일

- `docs-site/`에 Astro Starlight 스캐폴딩 + 의존성 설치(Starlight 0.40).
- `astro.config.mjs`: 제목·설명, 한국어 루트 로케일(ko-KR), **base `/kc`**, 9개 섹션 사이드바(자동생성).
- 컨벤션 문서 먼저 작성: [개요](/kc/start/overview/), [글쓰기 규칙](/kc/start/contributing/).
- 정보구조(시작/기획/리서치/아키텍처/의사결정/실험/기록/지식베이스/템플릿) + 각 섹션 인덱스.
- 템플릿 4종(리서치·ADR·실험·일지), 첫 ADR([0001](/kc/decisions/0001-astro-starlight/)).
- 기존 16문서 마이그레이션 진행.

## 배운 점 / 확정한 사실

- (검증됨) **Starlight 0.40에선 label 있는 자동생성 그룹을 `items: [{ autogenerate }]`로 감싸야** 한다(구형 직접형식은 에러).
- (검증됨) **base `/kc`면 마크다운 절대링크에 base가 자동 안 붙는다** — 빌드 출력 HTML이 `href="/kc/..."`로 나오므로, 작성 시에도 `/kc/...`로 써야 일치(빌드로 확인). dist는 **평탄**(dist/start/…)이라 API는 `/kc/<경로>`→`dist/<경로>`로 서빙.

## 막힌 점 / 열린 질문

- (미검증) sitemap은 `site` 옵션 없으면 스킵 — 내부용이라 보류.
