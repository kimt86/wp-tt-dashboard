**상태:** `진행중` · **최종 검토:** 2026-06-12

## 가설

레그(leg, topos1 타깃 단위) 기반 차세대 사이클 감지(v2)가 현행 이동필터(v1)보다 **사이클 경계와 trip별 도착을 더 정확하게** 잡을 수 있다. v1 핫패스는 건드리지 않고 **그림자(병렬)** 로 `tt_cycle_v2`에 적재해 같은 행(`ytno`+`dropped_at`)으로 페어드 비교한다.

### 수집 우선순위 (사용자 재정의)

1. **사이클 열림/닫힘 시각** (최우선)
2. **각 trip의 시작·도착 위치+시각** (차순위)
3. 크레인 핸드오버 시작/종료 (있으면 좋고 없으면 뺌)

## 방법 — 비교·게이트 규칙

- **페어드 비교**: 같은 `(ytno, dropped_at)` 행에서 v1 vs v2. 운영 변동(시간대별 도착율 출렁임)이 양쪽에 동일 작용.
- **워밍업 필터**: 재시작 직후 ~15분 내 완료 사이클은 앞 단계 누락 → `opened_at > 재시작+15분`만 평가.
- 수치엔 표본 수(n)와 측정창을 함께.

## 핵심 발견 (확정)

- (검증됨) **크레인(QC) 도착은 GPS로 못 잡는다** — QC GPS가 디바이스맵에 유효 진입하지 않아 `arr_src=gps`·`arrived`가 실측 거의 0. 약경계의 근본 원인.
- (검증됨) 그래서 크레인 도착 정확도는 **`cur_loc=WHARF`의 ~200초 조기 latch를 이후 `arr_dtime`/`ARRIVED`로 업그레이드**해서 얻는다(Fix A). 이게 |Δ| 200초→1초를 풀었다.
- (검증됨) **닫힘 `dropped_at` = container1 클리어**(≈인계 종료 + TOS 갱신, 드롭 도착 후 중앙 ~5.5분) — 물리적 들어올림 순간이 아니다. 외부 정답지 없음.
- (검증됨) v2는 **거짓 분할을 억제**해 경계가 v1보다 정확: 2분 미만 비정상 사이클 v1 7.5% → v2 0.9%, 중앙 길이 14→17.6분.

## 6-이벤트 모델 (현재)

핸드오버 *시작 시각*은 웹소켓으로 신뢰도 있게 관측 불가(QC PLC에 트럭 식별자 없음) → **수집 포기**. 받기·주기는 체류 간격으로 대체.

```
사이클시작 ─[공차이동]─ 공차완료 ─[받기]─ 부하시작 ─[부하이동]─ 부하완료 ─[주기]─ 사이클종료
           (공차이동시작)
```

| 이벤트 | 컬럼 | 관측 |
|---|---|---|
| ① 사이클 시작 | `opened_at` | 첫 배정 신호(topos1/container1) |
| ② 공차이동 시작 | `empty_travel_start_at` | 오픈 후 첫 출발. ①→②=배차 후 대기. 선배치면 NULL |
| ③ 공차이동 완료 | `empty_arrived_at` | 픽업 도착 |
| ④ 부하이동 시작 | `pickup_left_at` | 픽업 출발 = topos1 플립 |
| ⑤ 부하이동 완료 | `laden_arrived_at` | 드롭 도착 |
| ⑥ 사이클 종료 | `dropped_at` | container1 클리어(이동필터) |

받기 = ④−③, 주기 = ⑥−⑤ (체류).

## 반복 이력과 결과

| 버전 | 변경 | 핵심 결과 |
|---|---|---|
| v2.2 | 레그 트래커 기준선 | 크레인측 \|Δ\| ~200초, LD픽업 45% |
| **v2.3** | A(cur_loc→arr_dtime 업그레이드)·B(선배치)·C(조기종료 이월) | **\|Δ\| 200→1초**(G3 해결), 주순서위반 0 |
| **v2.4 / 4.1** | D(v1 블록도착 백필) + 단조 클램프 | 블록 포착 v1 추월(LD픽업 67%>64%, DS드롭 84%>80%), 단조위반 0 |
| **v2.5 / 5.1** | 6-이벤트 모델(핸드오버 제거, 공차이동시작 추가) | 전 경계 단조 0, 배차후 대기 중앙 24초 |
| **v2.6** | container1 픽업 레그 보장 | **픽업 도착 66→84%**, 드롭전용 1-레그 LD 211→6·DS 196→8 |

### 게이트 (설계)

- G1 약경계 포착 (DS픽업·LD드롭 절대 목표) · G2 강경계 ≥ v1−5%p · G3 \|Δ\| 중앙값 ≤ 30초 · G4 순서위반 0
- 현황: **G3·G4 통과**, 블록측 G2 통과(백필), G1 DS픽업 절대 60%는 크레인 신호 한계로 미달하나 v1을 +5~13%p 추월.

## 해석 / 다음

- 타이밍은 v1과 사실상 동일(1초), 경계는 v1보다 정확, 픽업 포착 84%. → 승격(메인 컬럼 v2 교체) 후보.
- **CYCLES 페이지에 6이벤트 노출 완료**(v1/v2 토글, 미관측=빈칸).

## 열린 질문

- (미검증) LD픽업 포착이 표본별 G2 경계에서 출렁(−4~+3%p) — 더 큰 표본(24h)으로 중심값 확정 필요.
- (미검증) 잔존 1-레그(~29%)는 주로 LD 픽업전용 = 크레인 드롭 도착 미관측. 관측 불가분이라 NULL 유지(허위 생성 금지).
- 승격 시점·기준(절대 G1 vs "v1 이상")을 확정할 것.
