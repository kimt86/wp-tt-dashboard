## 문제

컨테이너 터미널의 트랜스퍼 트럭(TT) 운영은 가시성이 낮다. 어떤 트럭이 어떤 작업을 얼마나 효율적으로 도는지, 어디서 대기·공차가 생기는지 **실시간·이력 모두 파악이 어렵다.** 배차는 경험에 의존한다.

## 사용자

- **운영/배차 담당** — 실시간 TT 상태·대기·병목을 본다.
- **분석가/엔지니어** — KPI·사이클 이력으로 효율을 측정하고 개선·모형을 설계한다.

## 범위 (현재)

- 실시간 모니터링 대시보드: TT 위치·분류(공차/배송/대기/staging), QC·블록 현황.
- 운영 KPI 7종 + websocket 실측 교차검증.
- **작업 사이클 이력**: 검증된 컨테이너 인도 단위로 적재·분석(학습 데이터 누적).
- 차세대 사이클 감지(v2 그림자) — 6-이벤트 모델로 trip별 시작·도착 포착.

## 범위 (다음 / 연구)

- **AI 배차** — 사이클·대기 예측 기반 추천. 현재 [예측 모형 연구](/kc/research/tt-prediction/) 단계.

## 비목표 (지금은 아님)

- 실제 자율주행 차량 제어. (모니터링·분석·추천이 우선)
- prod Oracle 직접 연동(추출기/게이트웨이만 사용).

## 성공 지표 (초안 — 검증 필요)

- (미검증) 사이클 감지가 운영 실측과 일치(타이밍 오차 작게, 포착률 높게).
- (미검증) 대기/공차 시간 가시화로 배차 개선 여지를 정량화.

## 열린 질문

- (미검증) AI 배차의 1차 목표 지표는? (회전율 / 공차 최소화 / QC 대기 최소화 중 무엇)
- 운영자가 매일 보는 단일 화면은 무엇이어야 하나.
